# 03 — KV v2 (версии и откат)

[← Все гайды](README.md)

## Цель

Хранить секреты с версионированием, читать старую версию и откатываться при ошибочной записи.

## Предварительные условия

- Работающий Tuck с root-токеном

> В Tuck KV v2 — отдельный API на пути `/v2/secret/` (не Vault-стиль `secret-v2/data/`).
> Политики используют тот же префикс `secret/` что и KV v1.

## Шаги

### 1. Записать секрет (версия 1)

Тело запроса — **сырые байты** (или текст):

```powershell
curl -k -X PUT "$env:TUCK_ADDR/v2/secret/myapp/config" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN" `
  -H "Content-Type: text/plain" `
  -d "v1-key-value"
```

Ответ: `{"version":1,"path":"myapp/config"}`.

### 2. Обновить (версия 2)

```powershell
curl -k -X PUT "$env:TUCK_ADDR/v2/secret/myapp/config" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN" `
  -H "Content-Type: text/plain" `
  -d "v2-key-value"
```

### 3. Прочитать текущую версию

```powershell
curl -k "$env:TUCK_ADDR/v2/secret/myapp/config" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN"
```

В ответе: `value` = `v2-key-value`, `version` = `2`.

### 4. Прочитать версию 1

```powershell
curl -k "$env:TUCK_ADDR/v2/secret/myapp/config?version=1" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN"
```

### 5. Метаданные (все версии)

```powershell
curl -k "$env:TUCK_ADDR/v2/secret/metadata/myapp/config" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN"
```

### 6. Мягкое удаление версии

```powershell
curl -k -X DELETE "$env:TUCK_ADDR/v2/secret/myapp/config?versions=1" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN"
```

### 7. Восстановить удалённую версию

```powershell
curl -k -X POST "$env:TUCK_ADDR/v2/secret/undelete/myapp/config" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN" `
  -H "Content-Type: application/json" `
  -d '{"versions":[1]}'
```

### 8. (Опционально) CAS — запись только если версия не изменилась

```powershell
curl -k -X PUT "$env:TUCK_ADDR/v2/secret/myapp/config?cas=2" `
  -H "X-Tuck-Token: $env:TUCK_TOKEN" `
  -d "v3-only-if-still-v2"
```

## Проверка

| Действие | Ожидаемый результат |
|----------|---------------------|
| GET без version | последняя версия |
| GET `?version=1` | `v1-key-value` |
| metadata | список версий с timestamps |
| soft-delete + undelete | версия снова читается |

## Политики

```json
[
  {"path": "secret/myapp/*", "capabilities": ["read", "write", "delete", "list"]}
]
```
