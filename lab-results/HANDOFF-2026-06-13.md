- Test 9b: роль `tuck/tuck-operator` с policy `root`, restart operator, удаление stale `demo-app-credentials`
- Reload образа: `minikube ssh "docker rmi -f tuck-server:local"` перед `image load` (иначе старый образ 2-дневной давности!)
- `Get-K8sRootToken`: валидация токена + fallback из логов pod
- `-p 1` для unit-тестов (меньше port-conflict)

## Уроки / подводные камни

1. **`minikube image load` не перезаписывает** образ с тем же тегом — нужен `minikube ssh "docker rmi -f tuck-server:local"` перед load.
2. **После сброса PVC** root token меняется; оператору нужна заново роль K8s auth: `PUT /v1/auth/kubernetes/role/tuck/tuck-operator` с `policies: ["root"]`.
3. **Test 9b** падал из-за stale K8s Secret `demo-app-credentials` (значение `rotated-key-xui` от прошлого прогона) + operator 403 без роли.
4. **Port-forward** может умереть после rollout — скрипт теперь проверяет health, не только listen на порту.
5. **README `lab-results/`** — кодировка/бэктики в markdown от `Set-Content` иногда ломаются; при правке использовать нормальный UTF-8.
